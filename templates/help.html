帮助

本网站由多个 Git 仓库中的 Markdown 文件聚合生成。

1. 仓库配置

项目根目录下的 config.json 最简形式只需要写远程仓库 URL，例如：

{
  "repos": [
    { "url": "git@github.com:you/tech-blog.git" },
    { "url": "https://github.com/you/life-notes.git" }
  ]
}
  • url：远程 Git 仓库地址，必填。
  • title（可选）：用于 URL 中的路径段，例如 /tech；如果不写，则自动使用仓库名（例如 tech-blog.git → tech-blog）。
  • dir（可选）：该仓库在当前项目中的本地目录，相对于项目根目录；如果不写，则默认 repos/<仓库名>。

2. 仓库结构

每个仓库目录内部约定为：title/*.md，例如：

project-root/
├── tech/          # 一个 Git 仓库工作目录
│   ├── post1.md
│   └── post2.md
└── life/          # 另一个仓库
    └── note1.md

3. 访问说明

  • /index：全站最新文章列表（聚合所有仓库）
  • /<title>：对应仓库（博客）的文章列表，例如 /tech、/life
  • /<title>/<slug>/：具体某篇文章，例如 /tech/post1/
  • /help：当前帮助页面

当 Git 仓库内容更新后，可通过配置好的 Git Hook 或 Webhook 调用 /webhook 接口触发重新构建。
