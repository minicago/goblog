帮助

本网站由单个 Git 仓库根目录下的 Markdown 文件生成，文件名作为标题。

1. 仓库配置

项目根目录下的 config.json 示例（只有一项）：

{
  "repos": [
    { "url": "git@github.com:you/tech-blog.git" }
  ]
}
  • url：远程 Git 仓库地址，必填。
  • dir（可选）：该仓库在当前项目中的本地目录；默认 repos/<仓库名>。

2. 仓库结构

服务器只会扫描仓库根目录下的 Markdown 文件，子目录会被忽略。例如：

project-root/
└── tech-blog/
    ├── post1.md
    ├── another.md
    └── subdir/

3. Markdown 文件格式

每个 Markdown 文件需要包含 metadata 代码段，格式如下：

---
title: 文章标题
date: 2024-01-01 12:00
category: 分类名称
difficulty: 难度等级
---
文章内容（支持 Markdown 语法）
  • title：文章标题，必填。
  • date：发布时间，必填，格式为 YYYY-MM-DD HH:MM。
  • category：文章分类，可选。
  • difficulty：文章难度等级，可选。

3. 访问说明

  • /index：文章列表
  • /<slug>/：具体某篇文章，例如 /post1/
  • /help：当前帮助页面

当 Git 仓库内容更新后，可通过配置好的 Git Hook 或 Webhook 调用 /webhook 接口触发重新构建。
